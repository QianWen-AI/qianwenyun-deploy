# 交互规范

## 排版约定

- `> 💬` 引用块 = 可直接展示给用户的话术
- `> ⚠️` 引用块 = Agent 必须遵守的技术约束
- 可用 ECS / EIP / SLB / RDS / 公网 IP 等开发者术语，但不出现内部黑话或无说明的代码

## 禁止行为

- **绝对不允许**建议用户在提示符前加 `!` 来运行命令。需要用户执行命令时，直接告知用户在独立终端中运行，不得提及 `!` 前缀方式。

## Fail-fast

步骤 11（CreateStack）之前必须确认：Docker 已装（若需要）、产物存在、库存有货、模板验证通过。不满足时给修复路径，**不得创建栈**。

## 额外资源费用透明

> ⚠️ 部署过程中 Agent 自行创建的任何产生费用的云资源（OSS Bucket、SLB、RDS 等），**必须在创建前明确告知用户并获得确认**，不得静默创建。

具体要求：

1. **全栈部署（步骤 9–10）**：`upload_artifacts.py` 会创建临时 OSS 桶存放构建产物。Agent 须在首次执行前告知用户：
   > 💬 部署过程中需要创建一个临时 OSS 桶来存放构建产物（桶名形如 `qianwenyun-deploy-tmp-xxx`）。该桶设有 7 天自动过期策略，删除部署时会一并清理。OSS 存储费用极低（约 ¥0.12/GB·月），产物通常不超过几百 MB。

2. **全栈部署（步骤 9 询价确认）**：询价汇总时必须列出本次部署将创建的**全部计费资源**（ECS、EIP、SLB、RDS、临时 OSS 桶等），不得遗漏。

3. **通用原则**：任何脚本如果会调用 `aliyun oss mb`、`CreateDBInstance`、`CreateLoadBalancer` 等创建计费资源的 API，Agent 在调用前必须先向用户说明将创建什么资源、预估费用、以及后续如何清理。

## 敏感配置安全提醒

> ⚠️ 部署上线前，Agent 必须提醒用户检查项目中是否存在不应暴露到生产环境的敏感信息。

在以下时机触发提醒：

1. **步骤 3（项目分析）后**：如果 `analyze_project.py` 的 `env_files` / `source_samples` / `config_files` 中检测到疑似硬编码的密钥、Token、连接串等敏感内容（如 `sk-xxx`、`AKID`、`password=`、`jdbc:mysql://user:pass@`），Agent 必须立即警告用户：
   > ⚠️ 检测到项目中可能存在硬编码的敏感信息（如 API 密钥、数据库密码等）。部署到公网后这些信息可能被泄露。请确认：
   > - 源码中的密钥/Token 是否已移到环境变量或 `.env` 文件（不入 git）
   > - `.env`、`config/secrets.*` 等敏感文件是否已加入 `.gitignore`
   > - 前端代码中是否残留了不应公开的 API Key（前端代码对用户可见）

2. **通用原则**：
   - 凡是将文件上传到公网可访问位置（ECS Nginx 静态目录），Agent 都须确认产物中不含敏感文件
   - 发现 `.env`（非 example/sample/template）出现在待上传目录中时，须**明确告知用户风险**，由用户决定是否继续

## 步骤展示

全栈部署共 14 个步骤，但并非每个项目都会经历全部步骤。Agent 在执行过程中须**始终展示完整步骤列表**，跳过的步骤标注「已跳过」并说明原因，让用户对整体进度一目了然。

展示格式示例：

> ✅ 步骤 1 · 环境检查 — 通过
> ⏭️ 步骤 2 · Git URL 处理 — 跳过（本地项目，无需 clone）
> ✅ 步骤 3 · 项目分析 — 完成
> ✅ 步骤 4 · 存量检测 — 无存量，继续新建
> ⏭️ 步骤 5 · 数据库识别 — 跳过（项目无数据库依赖）
> 🔄 步骤 6 · 拓扑 & 规格选择 — 进行中…

规则：
- 每完成一个步骤或跳过一个步骤时，向用户更新进度
- 跳过原因须简明（如「本地项目」「无数据库依赖」「非 Git URL」「纯前端无后端」等）
- 不要省略跳过的步骤号，保持编号连续，避免用户看到 1→3→6 产生疑惑
- 可在关键节点（如步骤 9 询价确认前）汇总展示一次全部步骤状态

## 过程感知

长等待开始前：`⏳ [操作]（预计 X 分钟）· 费用：[尚未/已开始] · 中断：[安全/需清理]`

创建栈期间每 60-90 秒播报心跳：`⏳ 仍在创建中，已等待 8 分钟…`。含 RDS 时提前说明：`数据库创建约 10-30 分钟，属正常`。

## 错误兜底

遇错必答三个问题：**发生了什么** · **资源/费用状态** · **下一步选项**。

- 🟢 可忽略：一句话，继续
- 🟡 可恢复：资源安全，给重试/检查/清理选项
- 🔴 需介入：告知资源状态（已回滚 / 需手动清理）

禁止：只说「失败」不给下一步、直接抛 API JSON、探活失败说「部署失败」。
